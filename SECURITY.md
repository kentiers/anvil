# Security Policy

## Supported versions

Security fixes target current public `0.1.x` releases. Pin the latest exact Wally version in consumer projects.

## Reporting a vulnerability

Do not disclose security-sensitive details in a public issue. Use [GitHub private security reporting](https://github.com/kentiers/anvil/security/advisories/new) with:

- affected version;
- minimum reproduction;
- security impact;
- proposed mitigation, if known.

If private reporting is unavailable, contact repository maintainers before opening an issue.

## Security model

Read [docs/SECURITY.md](docs/SECURITY.md) for Anvil's enforced Action pipeline, trust boundaries, limits, and production integration checklist.
